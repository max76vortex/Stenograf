# Учёт задач workspace

## Главный принцип

**Каждая осмысленная задача** получает **ID** и **одну строку** в реестре **`task-registry.md`**. Остальные файлы — дополняют, но не заменяют реестр.

---

## Где что фиксировать

| Что | Файл | Когда |
|-----|------|--------|
| **Реестр (ID, статус, даты)** | **`memory-bank/task-registry.md`** | При появлении задачи → строка `Open`; при закрытии → `Done` + дата завершения |
| **Крупные фазы проекта** | `memory-bank/tasks.md` | Чеклисты фаз; в пункте можно дать ссылку: «см. WS-012» |
| **Хронология «что сделали»** | `memory-bank/progress.md` | После значимого шага или закрытия задачи — **в конце строки укажи `[WS-NNN]`** |
| **Время/события (опционально)** | `memory-bank/activity-journal.md` | Ручная строка или хук после коммита |
| **Код / конфиги** | Git | В сообщении коммита желательно: `WS-NNN: кратко` |

### Идентификатор задачи: `WS-NNN`

- **`WS`** — **workspace** (данный репозиторий). При желании префикс можно заменить на другой в реестре и в коммитах (один раз зафиксируй в `task-tracking.md`).
- **`NNN`** — три цифры, порядковый номер. Следующий номер = **max + 1** в `task-registry.md`.

Примеры: `WS-001`, `WS-042`.

---

## Жизненный цикл

1. **Старт задачи** — строка в **`task-registry.md`** (Status `Open`, дата).
2. **В процессе** — детали в коде/артефактах; реестр обновляй по необходимости.
3. **Закрытие** — в реестре `Done`; в **`progress.md`** строка с **`[WS-NNN]`**; опционально журнал и коммит `WS-NNN:`.

---

## Что не считать отдельной задачей

Мелкие правки без смысловой вехи — по усмотрению. Порог: «если пишем отдельную строку в `progress.md` — заведи `WS-NNN`».

---

## Связанные файлы

- `memory-bank/task-registry.md` — источник истины по списку задач
- `memory-bank/tasks.md`, `progress.md`, опционально `activity-journal.md`
