# Глобальные настройки Cursor (для любого нового проекта)

Скилл **`memory-bank-init`** создаёт файлы **внутри репозитория**. Чтобы агент **во всех workspace** помнил про `WS-NNN` без копирования `.cursor/rules/`, добавь короткое правило **на уровне пользователя**.

## Куда нажать (Cursor)

1. Открой **Cursor Settings** (шестерёнка) → раздел **Rules** / **General** → **User Rules** (или **Rules for AI** — формулировка зависит от версии).
2. Либо файл **`.cursorrules`** в домашней папке пользователя — **только если** ты им уже пользуешься (не все версии Cursor читают его одинаково).

Приоритет: то, что явно указано в **User Rules** в UI.

## Что вставить (минимум)

Скопируй блок ниже в **User Rules**:

```markdown
### Memory Bank — учёт задач (WS-NNN)

Если в корне workspace есть папка `memory-bank/` и файл `memory-bank/task-registry.md`:
- при появлении значимой задачи — новая строка в реестре с ID **WS-NNN** (следующий номер после максимального в таблице);
- при закрытии — статус Done, дата; в `memory-bank/progress.md` в строке итога добавь **`[WS-NNN]`**;
- коммиты по возможности с префиксом **`WS-NNN:`**.
Полная схема: `memory-bank/task-tracking.md` (создаётся скиллом инициализации).
```

## Расширенный вариант

Если не хочешь дублировать текст, в User Rules достаточно:

```markdown
Соблюдай `memory-bank/task-tracking.md` и `memory-bank/task-registry.md`, если они есть в проекте.
```

Тогда **обязательно** запускай скилл **memory-bank-init** в каждом новом репозитории, чтобы эти файлы появились.

## Не глобально

- **Скиллы** из `.cursor/skills/` лежат в репозитории — их копируют в новый проект или держат эталон в одном репо.
- **MCP, Obsidian** — отдельные настройки; не смешивай с учётом задач.
