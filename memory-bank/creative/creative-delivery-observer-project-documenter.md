# Creative — Delivery Observer / Project Documenter (draft for future start)

## Контекст

- **Проблема:** после завершения прогона артефакты разбросаны по разным файлам,
  а итоговая документация требует ручной сборки и проверки полноты.
- **Цель:** выделить отдельного наблюдателя-документатора, который отслеживает
  ключевые этапы и автоматически формирует консистентный пакет документации по
  проекту и прогону.
- **Базовый шаблон:** `memory-bank/archive/Архив - мультиагентная система Variant B.md`.
- **Ограничение текущего этапа:** только подготовка к реализации, без кода.

## Scope (что входит / что не входит)

- **Входит:**
  - наблюдение за ключевыми агентами (`01` + review/dev-контуры);
  - сбор и индексация артефактов run;
  - генерация итогового документационного пакета (summary/handover/reader guide);
  - проверка минимальной полноты и связности ссылок.
- **Не входит (позже):**
  - публикация внешней документации в сторонние системы;
  - генерация бизнес-контента вне контекста run-артефактов;
  - полная замена ручного editorial review.

## Роль в системе

- **Delivery Observer / Project Documenter** — специализированный субагент,
  который не принимает технические решения за доменные роли, а:
  - фиксирует доказательства выполнения этапов;
  - собирает единый narrative «что и почему сделано»;
  - готовит handover-пакет для нового участника/оператора.

## Входные данные

- `multi-agent-system/status.md`
- `multi-agent-system/current-run/technical_specification.md`
- `multi-agent-system/current-run/architecture.md`
- `multi-agent-system/current-run/plan.md`
- `multi-agent-system/current-run/reviews/*.md`
- `multi-agent-system/current-run/reports/*.md`
- `multi-agent-system/current-run/open_questions.md`
- Архивные/эталонные документы из
  `memory-bank/archive/Архив - мультиагентная система Variant B.md`

## Выходные артефакты (MVP)

- `multi-agent-system/current-run/reports/delivery_observer_summary.md`
  - статус этапов, ключевые решения, риски, открытые вопросы.
- `multi-agent-system/current-run/reports/project_handover_draft.md`
  - кому передаём, что сделано, что осталось, где доказательства.
- `multi-agent-system/current-run/reports/project_reader_guide_draft.md`
  - с чего читать проект, как воспроизвести, какие runbook использовать.

> На этапе архивации эти черновики должны легко маппиться на
> `final_summary.md`, `mas_handover_report.md`, `reader_guide.md`.

## Acceptance Checks для документатора

- **Полнота:** все обязательные этапы pipeline отражены в summary.
- **Трассируемость:** на каждое ключевое утверждение есть ссылка на артефакт.
- **Согласованность:** статусы в документах не противоречат `status.md`.
- **Практичность handover:** новый участник может продолжить работу по reader guide
  без дополнительных устных пояснений.
- **Ясность рисков:** явно перечислены технические и процессные риски.

## Оркестрационный flow интеграции (v1)

- После `Plan review` — первый снимок документации (baseline).
- После `Development` — обновление фактической реализации и тестов.
- После `Code review` — финализация блоков quality gate и unresolved points.
- Перед `Final summary` — генерация итогового документационного пакета.

## Policy качества (v1)

- Документатор не «придумывает» факты: только подтверждённые артефактами данные.
- Если не хватает данных для связного отчёта:
  - фиксируется gap-list;
  - статус документатора помечается как `NEEDS_INPUT`;
  - оркестратор останавливает финализацию до закрытия gap-list.
- Конфликты между источниками фиксируются отдельным блоком `Inconsistencies`.

## Риски и допущения

- **Риск:** дублирование с ручной архивацией.
- **Смягчение:** documenter готовит структурированный draft, архиватор делает
  финальную валидацию/публикацию.
- **Риск:** «галлюцинации» при неполных входных данных.
- **Смягчение:** строгая трассировка claim -> source artifact.
- **Риск:** перегрузка объёмом текста.
- **Смягчение:** шаблонные секции + лимиты на размер разделов + чеклист полноты.

## MVP на реализацию (когда стартуем)

- **MVP-1:** сбор артефактов и генерация `delivery_observer_summary.md`.
- **MVP-2:** генерация `project_handover_draft.md` и `project_reader_guide_draft.md`.
- **MVP-3:** интеграция в orchestrator checkpoints + quality policy hooks.

## Критерии приёмки будущей реализации

- На тестовом run формируется полный пакет из 3 документов без ручной сборки.
- Не менее 90% ключевых утверждений имеют явные ссылки на исходные артефакты.
- При missing artifacts документатор корректно ставит `NEEDS_INPUT`, а не
  выпускает «ложно полный» отчёт.
- На реальном run пакет принимается оператором как пригодный для handover.
