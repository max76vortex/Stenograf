# Creative — ASR benchmark protocol (RU, quality-gated)

## Контекст

- Текущий Phase A на Whisper даёт нестабильное качество на части записей
  (повторы, галлюцинации, слабая связность).
- Массовый прогон архива нельзя продолжать без quality gate.
- Нужен воспроизводимый выбор: **primary ASR + fallback ASR** для русской речи.

## Цель протокола

За 1-2 итерации тестов выбрать:

- **Primary engine** для ежедневного потока;
- **Fallback engine** для сложных ассетов;
- понятный auto/manual trigger перехода на fallback.

## Кандидаты для сравнения (минимум 3)

1. Текущий baseline: `faster-whisper` (наш pipeline).
2. RU-ориентированный сервис (например SpeechKit или аналогичный).
3. Второй облачный STT (Google/Azure/Deepgram/локальный сервис с API).

> Примечание: сравниваем не маркетинг, а качество на наших реальных файлах.

## Тестовый набор (golden set)

Минимум 10 файлов:

- 3 "хороших" (чистый голос),
- 4 "средних" (фон/эхо),
- 3 "тяжёлых" (перебивки, шум, длинные лекционные фрагменты).

Для каждого файла фиксируем:

- длительность,
- формат/размер,
- уровень шума (low/med/high),
- expected topic (1 строка, вручную).

## Метрики и пороги (quality gate)

### Блок A — критические дефекты (must-pass)

- **Looping/duplicates:** не более 1 повторяющегося блока > 8 слов на 5 минут.
- **Пустой/обрывочный вывод:** не допускается.
- **Смысловая связность:** минимум 70% предложений читаются как осмысленные.

Если провал по блоку A -> файл считается fail для движка.

### Блок B — эксплуатационное качество

- **Post-edit time:** сколько минут ручной правки на 10 минут аудио.
- **Пунктуация/абзацы:** субъективно 1..5.
- **Точность терминов/имён:** субъективно 1..5.
- **Стабильность на длинных файлах:** pass/fail.

## Шкала принятия решения

- Primary должен пройти:
  - >= 80% файлов по блоку A;
  - средний post-edit <= 12 мин / 10 мин аудио.
- Fallback должен:
  - улучшать минимум 50% файлов, которые primary провалил по блоку A.

Если никто не проходит порог -> повторный раунд после изменения preprocessing.

## Правило auto-fallback (после выбора движков)

Primary output отправляется в fallback, если:

- обнаружены повторы блока > 8 слов >= 2 раз, или
- длина текста подозрительно мала для длительности, или
- `needs_review=true` + low coherence по быстрой эвристике.

## План внедрения

1. Подготовить `golden set` и таблицу.
2. Прогнать 3 кандидата.
3. Заполнить таблицу, выбрать primary/fallback.
4. Обновить Phase A runbook и команду запуска.
5. Только после этого продолжать `WS-010` (массовый прогон).
