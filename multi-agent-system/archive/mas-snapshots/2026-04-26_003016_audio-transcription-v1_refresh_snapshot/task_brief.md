# Task Brief

## Задача

Реализовать и довести до практического использования **конвейер транскрибации** накопленных диктофонных записей (~30 ГБ mp3): локально, максимальное качество (Whisper large-v3), вывод в отдельный vault Obsidian **Audio Brain** в папку `00_inbox` с предсказуемым порядком и без путаницы между исходниками и транскриптами.

**MAS Project ID:** `audio-transcription-v1` (slug подобран по смыслу: аудио → текст → Obsidian; для будущих прогонов MAS slug назначается по теме проекта).

## Цель

- На выходе — **готовые заметки-транскрипты** в Obsidian (`.md` с frontmatter и блоком «Транскрипт»), однозначно связанные с исходным mp3 (`audio_file`).
- **Порядок:** структура папки записей (`recordings/YYYY-MM/`), шаблон имён mp3 (`YYYY-MM-DD_NNN[_метка].mp3`), плоский `00_inbox`, возможность повторного запуска без перезаписи уже готовых файлов.
- **Документация и скрипты** в репозитории: установка на ПК (SETUP), запуск, опционально манифест и проверка покрытия.

## Критерии успеха

- Установлено окружение Python + faster-whisper; скрипт `transcription/transcribe_to_obsidian.py` успешно прогоняет тестовую папку mp3 и создаёт .md в `00_inbox`.
- Пользователь может по README/SETUP воспроизвести установку и запуск на своей машине (пути к `recordings`, vault Audio Brain).
- Описан и при необходимости автоматизирован контроль «какие mp3 ещё без .md» (`check_coverage.py`).
- **Не входит в этот прогон:** постинг в Ghost, синхронизация статей с CMS (отдельная фаза после появления материалов к публикации).

## Scope

- **Входит:** faster-whisper large-v3, скрипт транскрибации, `--recursive`, `--manifest`, документация README/SETUP, пример bat, утилита `check_coverage.py`, дизайн порядка в `memory-bank/creative/creative-transcription-workflow.md`, шаблоны и структура vault Audio Brain (уже созданы ранее).
- **Не входит:** интеграция Obsidian ↔ Ghost, n8n-потоки постинга, правка контента транскриптов вручную/LLM (это следующий этап работы пользователя).

## Ограничения

- Железо: GPU NVIDIA 6+ ГБ VRAM (RTX 3060), CUDA Toolkit для `device=cuda`; при необходимости fallback `--device cpu` (медленнее).
- Модель large-v3 качается при первом запуске; объём диска под кэш модели.
- Пути на диске пользователя: примеры `D:\1 ЗАПИСИ ГОЛОС\recordings\`, `D:\Obsidian\Audio Brain\00_inbox\` — не захардкожены в скрипте, только в документации.
