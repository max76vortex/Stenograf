# Архив прогонов MAS

**MAS** — сокращение от **M**ulti-**A**gent **S**ystem (`multi-agent-system/`).

## Зачем эта папка

Завершённый прогон через MAS **не считается закрытым**, пока выполнена **обязательная архивация**: снимок артефактов и отчётов переносится в структурированное хранилище с **уникальным именем проекта**, чтобы разные запуски не смешивались.

Принцип: *конец — всему делу венец* — финал включает не только `Final summary` в `current-run/`, а **полный пакет знаний** о проекте для любого, кто откроет архив позже.

## Идентификатор проекта MAS (`mas_project_id`)

- Короткое **уникальное** имя в стиле **slug**: латиница, цифры, дефисы, например: `n8n-pipelines-v1`, `dry-run-status-surface`.
- Задаётся **до или в начале** прогона и фиксируется в `current-run/project_context.md` (строка вроде `**MAS Project ID:** \`...\``).
- По этому slug строятся **папки в архиве workspace** и **папки/заметки в Obsidian**.

## Структура в workspace

Рекомендуемый путь для копии завершённого прогона:

```text
multi-agent-system/archive/mas-projects/<mas_project_id>/runs/<YYYY-MM-DD>/
```

Внутри — копии ключевых артефактов из `current-run/` на момент закрытия, плюс обязательные файлы отчёта (см. `runbooks/mas-archive-policy.md`).

Допустим вариант с суффиксом прогона, если за день несколько: `YYYY-MM-DD_run02`.

## Копия в Obsidian

Параллельно с архивом в репозитории создаётся **зеркальная** документация в vault (см. `runbooks/mas-archive-policy.md`): тот же `mas_project_id`, исчерпывающее описание проекта и инструкции для читателя.

**Папка в vault (Audio Brain)** — документация MAS хранится в **`20_system/Система мультиагентов/`** (см. `.cursor/rules/obsidian-content-boundaries.mdc`), не в `Base/`, не в `ИТ проекты/`, не в inbox/processed.

Подробности — в `multi-agent-system/runbooks/mas-archive-policy.md`.
