# Technical Specification

## 1. Общее описание и цель
Разработка и автоматизация локального конвейера транскрибации аудиозаписей (mp3) в текстовые заметки Obsidian (vault Audio Brain). Цель текущего этапа — доведение системы до полностью автоматического режима (fully automatic) с внедрением абстракции провайдеров распознавания речи (ASR provider abstraction) в основном потоке разработки (Core/MAS Delivery, Worktree 1), в то время как исследования и бенчмаркинг ASR-движков вынесены в параллельный поток (ASR R&D, Worktree 2).

## 2. Область изменений в существующей системе
- Внедрение абстракции ASR-провайдеров в `transcribe_to_obsidian.py` (замена жесткой привязки к faster-whisper на гибкий интерфейс).
- Доработка скриптов транскрибации и проверки покрытия (`check_coverage.py`) для поддержки полностью автоматического режима работы.
- Обновление документации (README, SETUP) с учетом новых абстракций и автоматического режима.
- Интеграция результатов бенчмарков из Worktree 2 (ASR R&D) через зафиксированный интерфейс.

## 3. Use case

**Название:** Автоматическая транскрибация новых аудиозаписей
**Актеры:** Пользователь, Система (Скрипт транскрибации)

**Предусловия:**
- Установлено окружение Python 3.10+ и необходимые зависимости.
- Настроены пути к папке с аудиозаписями (`recordings/YYYY-MM/`) и vault Obsidian (`00_inbox/`).
- Выбран и настроен ASR-провайдер через абстрактный интерфейс.

**Основной сценарий:**
1. Пользователь или планировщик запускает скрипт транскрибации (например, через `.bat` файл).
2. Система сканирует целевую папку с аудиозаписями рекурсивно.
3. Система определяет файлы, для которых еще нет соответствующих `.md` заметок в Obsidian (используя логику `check_coverage.py`).
4. Для каждого нового файла система вызывает выбранного ASR-провайдера через интерфейс абстракции.
5. ASR-провайдер выполняет распознавание речи и возвращает текст.
6. Система формирует `.md` файл с frontmatter и блоком транскрипта.
7. Система сохраняет файл в папку `00_inbox` vault'а Audio Brain.

**Альтернативные сценарии:**
- *Отсутствие новых файлов:* Система сообщает, что все файлы уже транскрибированы, и корректно завершает работу.
- *Ошибка ASR-провайдера:* Система логирует ошибку для конкретного файла и переходит к обработке следующего.
- *Отсутствие GPU (если выбран локальный провайдер, требующий GPU):* Система переключается на CPU (fallback), предупреждая пользователя о снижении скорости.

**Постусловия:**
- В папке `00_inbox` появляются новые `.md` заметки для всех ранее не обработанных аудиозаписей.
- Исходные аудиофайлы остаются неизменными.

**Критерии приемки:**
- Скрипт успешно обрабатывает тестовую директорию с mp3 файлами.
- Созданные `.md` файлы имеют корректный формат, frontmatter и содержат распознанный текст.
- Повторный запуск скрипта не перезаписывает и не дублирует уже существующие транскрипты.
- Скрипт поддерживает смену ASR-провайдера через конфигурацию/интерфейс без изменения основной бизнес-логики.

## 4. Нефункциональные требования
- **Производительность:** Поддержка работы на GPU (NVIDIA 6+ ГБ VRAM) для локальных моделей (например, faster-whisper large-v3) с возможностью fallback на CPU.
- **Масштабируемость:** Способность обрабатывать большие объемы данных (~30 ГБ mp3) порциями или в пакетном режиме.
- **Расширяемость:** Архитектура должна позволять легко добавлять новые ASR-движки (через ASR provider abstraction).
- **Надежность:** Устойчивость к сбоям при обработке отдельных файлов (ошибка в одном файле не должна прерывать весь процесс пакетной обработки).

## 5. Ограничения и допущения
- **Ограничения:**
  - Пути к директориям (`recordings`, `00_inbox`) не должны быть жестко закодированы в скриптах.
  - Интеграция с Ghost и n8n-потоки постинга находятся вне рамок текущей задачи.
  - Модель large-v3 (при использовании faster-whisper) скачивается при первом запуске, требуется достаточное место на диске.
- **Допущения:**
  - Структура имен файлов mp3 соответствует шаблону `YYYY-MM-DD_NNN[_метка].mp3`.
  - Worktree 2 (ASR R&D) предоставит готовое решение/бенчмарк для интеграции в Core поток через согласованный интерфейс.
