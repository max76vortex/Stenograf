# Политика обязательной архивации MAS

**MAS** = `multi-agent-system` (мультиагентная система).

Архивация завершённого прогона — **обязательный** этап, не опция. Без неё цикл «проект через MAS» считается незавершённым с точки зрения передачи знаний.

## Принцип

> Конец — всему делу венец.

После `Final summary` и статуса `COMPLETE` в `status.md` нужно:

1. **Итог по этапам** — уже есть в `current-run/reports/final_summary.md` (и смежных отчётах).
2. **Полный отчёт о проделанной работе** — что сделано по сути, решения, артефакты, ссылки на файлы в репозитории.
3. **Инструкции для читателя** — как устроен проект, как воспроизвести, что проверить, куда смотреть, известные ограничения.
4. **Идентификация проекта** — обязательный **`mas_project_id`** (slug), чтобы разделять проекты в архиве.

## Идентификатор `mas_project_id`

- Формат: `kebab-case`, латиница, цифры, дефисы.
- Пример: `audio-brain-pipeline`, `n8n-workspace-mas-v1`.
- Фиксируется в `project_context.md` (рекомендуемая строка в начале или в блоке метаданных):

```markdown
**MAS Project ID:** `your-project-slug`
```

Без уникального slug архивные папки разных прогонов смешиваются — **не допускается** для новых проектов.

## Следующий прогон после архивации

Чтобы не смешивать артефакты, перед новой задачей с **новым** `mas_project_id` используйте скрипт чистого старта: `multi-agent-system/tools/mas-new-run.ps1` (скилл Cursor **mas-clean-start**) — снимок `current-run/` в `archive/mas-snapshots/`, сброс `status.md`, заготовки входных файлов.

## Что положить в архив workspace

Базовый каталог (создать при закрытии прогона):

`multi-agent-system/archive/mas-projects/<mas_project_id>/runs/<YYYY-MM-DD>/`

Рекомендуемое содержимое:

| Содержимое | Назначение |
|------------|------------|
| Копия `current-run/reports/final_summary.md` | Краткий финал этапов |
| `mas_handover_report.md` (новый файл) | Подробный отчёт: цели, что сделано, ключевые решения, риски, ссылки на код и runbook |
| `reader_guide.md` (новый файл) | Для нового читателя: контекст проекта, структура репо, как запускать, что читать первым |
| Копии или ссылки-пути на ключевые артефакты | По объёму: `technical_specification.md`, `architecture.md`, `plan.md`, важные `reviews/`, `reports/` |

Имена `mas_handover_report.md` и `reader_guide.md` можно объединить в один файл, если так удобнее, но **содержание** обоих типов должно быть исчерпывающим.

## Копия в Obsidian (обязательно)

Дублируй смысл архива в vault **этого** workspace (Audio Brain). Путь задаётся **локальным** правилом `.cursor/rules/obsidian-content-boundaries.mdc`, а не общим разделом `Base/` из глобальных правил.

### Куда сохранять

- **Каталог документации MAS в vault:** **`20_system/Система мультиагентов/`**
- Рекомендуемая структура для зеркала прогона:  
  **`20_system/Система мультиагентов/Архив прогонов/<mas_project_id>/`**  
  **или** одна заметка рядом, например:  
  **`20_system/Система мультиагентов/MAS — <краткое-русское-имя> — передача проекта.md`**

Имя файла — **по правилам Obsidian:** человекочитаемое, **на русском**, без даты в имени; slug `mas_project_id` удобно дублировать внутри заметки или подзаголовком.

**Не класть** служебную документацию MAS в `00_inbox`, `10_processed` или «в корень» vault — см. регламент:  
`20_system/Система мультиагентов/Регламент — разделение контуров работ.md` (приоритет при конфликте инструкций).

### Как сохранять (шаблоны — обязательно)

По глобальным правилам Obsidian **каждая новая заметка** создаётся на базе шаблона из папки шаблонов. В vault **Audio Brain** шаблоны лежат в **`20_system/templates/`** (не в корневой `Templates/` — её в этом vault нет, `obsidian_list_notes` по `Templates` вернёт пусто).

Содержимое `20_system/templates/` (на момент проверки): `Шаблон идеи.md`, `Шаблон проекта.md`, `Шаблон транскрипта.md`, `Шаблон черновика статьи.md`. **Зеркала архива прогона MAS** оформлять по **`Шаблон проекта.md`**: подставить дату закрытия и название вместо `{{date}}` / `{{project_name}}`; в **«Суть и ценность»** отразить проблему, решение, аудиторию и идентификаторы (`mas_project_id`, путь к архиву в репо); в **«Материалы из аудио»** при отсутствии записей указать «не применялись» и сослаться на `task_brief` / отчёты в архиве; в **«Следующие шаги»** — чеклист (в т.ч. смена `current-run/` перед новым прогоном).

1. `obsidian_list_notes` по пути **`20_system/templates`** — выбрать шаблон.
2. `obsidian_read_note` — прочитать шаблон.
3. Заполнить разделы под handover / reader guide.
4. `obsidian_update_note` с `targetIdentifier` вида `20_system/Система мультиагентов/...`

Верхний заголовок в теле заметки — **`##`**, не `#` (см. правила содержимого).

**Минимум по содержанию:**

- Идентификатор `mas_project_id` (в тексте или frontmatter).
- Ссылка на репозиторий: `N8N-projects`, путь `multi-agent-system/archive/mas-projects/<mas_project_id>/`.
- Суть отчётов `mas_handover_report` / `reader_guide`.
- Дата закрытия прогона.

## Чеклист закрытия (обязательный)

- [ ] `status.md` — `COMPLETE`, этапы подтверждены.
- [ ] В `project_context.md` задан **`MAS Project ID`** (для этого прогона).
- [ ] Создан каталог `archive/mas-projects/<mas_project_id>/runs/<YYYY-MM-DD>/`.
- [ ] Заполнены `mas_handover_report.md` и `reader_guide.md` (или эквивалентный объём в одном файле).
- [ ] Скопированы/перечислены ключевые артефакты `current-run/`.
- [ ] В **Obsidian** создана заметка (или папка) в **`20_system/Система мультиагентов/`** с тем же `mas_project_id`, по шаблону из **`20_system/templates/`** (не в `Base/`, не в inbox/processed).
- [ ] `current-run/` очищен или перенесён только **после** копирования в архив (по `current-run/README.md`).
