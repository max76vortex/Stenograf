# План разработки (Delta v1.2)

## 1. Этапы выполнения

1. **Сначала каркас и заглушки:** Выделение общей логики именования (Naming Layer) и создание базовых интерфейсов абстракции ASR (ASR Provider Abstraction).
2. **Затем интеграция:** Реализация локального провайдера `faster-whisper-local` и его внедрение в основной скрипт транскрибации `transcribe_to_obsidian.py`.
3. **Затем детальная реализация:** Доработка логики пакетной обработки (continue-on-error) и расширение полей манифеста для поддержки метаданных ASR.
4. **Затем укрепление тестов и документации:** Покрытие логики именования тестами (parity tests), актуализация эксплуатационной документации (`README.md`, `SETUP.md`) и фиксация контракта Benchmark Gate.

## 2. Список задач в порядке реализации

1. `task_01.md`: Создание абстракции ASR-провайдера и выделение слоя именования.
2. `task_02.md`: Реализация провайдера `faster-whisper-local` и интеграция в Core.
3. `task_03.md`: Реализация устойчивой пакетной обработки (continue-on-error) и обновление манифеста.
4. `task_04.md`: Написание тестов на паритет именования и smoke-тестов.
5. `task_05.md`: Обновление документации и фиксация контракта Benchmark Gate.

## 3. Зависимости между задачами

- **Task 01** не имеет зависимостей (каркас).
- **Task 02** зависит от **Task 01** (использует интерфейсы ASR).
- **Task 03** зависит от **Task 02** (обрабатывает ошибки, возвращаемые провайдером).
- **Task 04** зависит от **Task 01** (тестирует слой именования) и **Task 02** (smoke-тесты).
- **Task 05** зависит от **Task 02** и **Task 03** (документирует итоговое поведение).

## 4. Привязка задач к use case

- **UC-01 (Установка окружения):** Task 02, Task 05
- **UC-02 (Транскрибация папки месяца):** Task 01, Task 02, Task 03
- **UC-03 (Рекурсивная транскрибация дерева):** Task 01, Task 02, Task 03
- **UC-04 (Проверка покрытия транскрибации):** Task 01, Task 04
- **UC-05 (Повторный запуск после прерывания):** Task 03
- **UC-06 (Delta-контур Core и ASR R&D):** Task 01, Task 05

## 5. Отдельные задачи на тесты и документацию

- **Тесты:** Выделены в отдельную задачу **Task 04** (проверка паритета именования между `transcribe_to_obsidian.py` и `check_coverage.py`, а также smoke-тесты CLI).
- **Документация:** Выделена в отдельную задачу **Task 05** (обновление `README.md`, `SETUP.md` и описание правил приема новых ASR-движков через Benchmark Gate).
