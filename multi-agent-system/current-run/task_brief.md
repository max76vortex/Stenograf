# Dry Run Task Brief

## Task

Добавить в проект служебную страницу или команду, которая показывает:

- текущий статус системы;
- наличие открытых блокирующих вопросов;
- список последних созданных артефактов активного прогона.

## Dry Run Goal

Задача выбрана как первый dry run мультиагентной системы, потому что она:

- проходит полный контур аналитики, архитектуры и планирования;
- не требует большого объема кода;
- проверяет, умеет ли система работать с артефактами, статусом и подтверждением
  по этапам.

## Constraints

- нельзя ломать существующую инфраструктуру проекта;
- нельзя вводить тяжелую новую инфраструктуру без необходимости;
- нужно максимально использовать уже созданный контур `multi-agent-system/`.

## Expected Outcome

- понятный пользовательский сценарий просмотра статуса dry run;
- минимальная, но реализуемая архитектура;
- детальный план задач сверху вниз;
- возможность затем перейти к разработке по task-файлам.
