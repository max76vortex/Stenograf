# Task 03: Реализация устойчивой пакетной обработки (continue-on-error) и обновление манифеста

## 1. Цель задачи

Доработать логику пакетной транскрибации так, чтобы ошибки обработки отдельных файлов (например, сбой ASR-провайдера) не прерывали выполнение всего батча. Реализовать continue-on-error с финальным ненулевым кодом возврата, а также расширить манифест новыми полями для отслеживания ASR-статуса.

## 2. Связь с use case

- **UC-02:** Транскрибация папки месяца (продолжение после ошибки).
- **UC-03:** Рекурсивная транскрибация дерева (продолжение после ошибки).
- **UC-05:** Повторный запуск после прерывания (идемпотентность и статус).

## 3. Конкретные файлы изменений

- **Изменяются:**
  - `transcription/transcribe_to_obsidian.py`
  - `transcription/manifest.py` (если логика манифеста вынесена, иначе `transcribe_to_obsidian.py`)

## 4. Описание добавляемых или изменяемых классов, методов, функций

- **В `transcription/transcribe_to_obsidian.py`:**
  - Блок обработки файла (внутри цикла):
    - Обернуть вызов `provider.transcribe(request)` в `try...except AsrError`.
    - При ошибке `AsrError`:
      - Залогировать ошибку с указанием категории и `provider_id`.
      - Не создавать выходной `.md` файл.
      - Записать строку в манифест с `asr_status="error"` и `error_category`.
      - Установить флаг `has_errors = True`.
    - При успехе:
      - Записать `.md` файл.
      - Записать строку в манифест с `asr_status="success"`.
  - В конце скрипта:
    - Если `has_errors == True`, завершить скрипт с `sys.exit(1)`.
- **В логике записи манифеста (CSV):**
  - Добавить новые append-only колонки: `asr_provider`, `asr_model`, `asr_status`, `error_category`, `elapsed_sec`.
  - Сохранить совместимость со старыми колонками.

## 5. Тест-кейсы

- **TC-03.1:** Запуск батча с одним битым `.mp3` (вызывающим `AsrError`) и одним валидным `.mp3` обрабатывает оба файла, создает `.md` только для валидного, записывает обе строки в манифест и завершается с кодом `1`.
- **TC-03.2:** Запуск батча с полностью валидными файлами завершается с кодом `0`.
- **TC-03.3:** Манифест содержит новые колонки `asr_provider` и `asr_status` после успешного или ошибочного прогона.

## 6. Критерии приемки

- [ ] Ошибка транскрибации одного файла не прерывает обработку остальных файлов в папке.
- [ ] При наличии хотя бы одной ошибки в батче скрипт завершается с кодом `1`.
- [ ] Для файлов с ошибками не создаются частичные или пустые `.md` файлы.
- [ ] В манифест (если включен) записываются новые колонки `asr_provider`, `asr_status` и `error_category`.
- [ ] Существующие `.md` файлы не перезаписываются без флага `--overwrite`.
